# 3 分钟演示脚本

## 目标

展示这个项目不是普通 CRUD，而是一个能做经营诊断、数据归因、工具调用和审批闭环的电商运营 Agent。

## 演示步骤

1. **注册登录**
   - 打开 `http://127.0.0.1:5173/`
   - 注册一个新账号并登录。
   - 说明：开发环境不依赖真实内部系统，账号进入本地 demo store。

2. **运营驾驶舱**
   - 进入“运营驾驶舱”。
   - 展示 GMV、订单数、转化率、客单价、退款率、广告 ROI。
   - 指向“GMV 归因”，说明系统把 GMV 波动拆成流量、转化率、客单价三个因子。

3. **Agent 分析**
   - 进入“运营 Agent”。
   - 输入：`昨天 GMV 为什么下降？`
   - 展示 Agent 执行轨迹：
     - 读取经营指标快照
     - 拆解 GMV 变化归因
     - 识别经营异常
     - 商品分层与风险排序

4. **数据证据与建议动作**
   - 展示证据表格。
   - 说明建议不是拍脑袋生成，而是绑定具体指标和规则。
   - 展示高风险建议，例如暂停低 ROI 广告计划。

5. **建议审批**
   - 进入“建议审批”。
   - 审批一条待处理建议。
   - 说明企业级 Agent 不应自动执行高风险动作，需要人机协同闭环。

6. **商品与活动策略**
   - 进入“商品分析”，展示 ABC 分层、库存周转、风险标签。
   - 进入“活动策略”，展示主推商品和清仓商品策略。

## 结束语

这个项目展示了从数据建模、指标分析、Agent 工具编排、前端可视化到审批闭环的完整链路。即使没有真实电商平台，也能证明 AI Agent 和数据分析能力。
